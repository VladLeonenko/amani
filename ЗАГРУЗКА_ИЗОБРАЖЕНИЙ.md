# Инструкция по загрузке изображений товаров

## Проблема
Изображения товаров ссылаются на `http://localhost:3845`, который недоступен. Изображения не отображаются на сайте.

## Решение

### Вариант 1: Загрузка через админ панель (рекомендуется)

1. Откройте админ панель: `http://localhost:5173/admin/products`
2. Откройте каждый товар для редактирования
3. В разделе "Изображение" нажмите "Загрузить изображение"
4. Выберите файл изображения
5. Сохраните товар

### Вариант 2: Автоматическая обработка из папки

1. **Поместите изображения в папку:**
   ```bash
   frontend/public/products/
   ```

2. **Назовите файлы по slug товаров:**
   - `африканская-маска-дух-предков.jpg`
   - `картина-саванна-на-закате.jpg`
   - `скульптура-воин-масаи.jpg`
   - `постер-африканские-ритмы.jpg`
   - `фотография-жирафы-в-саванне.jpg`
   - `картина-африканская-деревня.jpg`
   - `скульптура-мать-и-дитя.jpg`
   - `картина-золотой-закат.jpg`
   - `фотография-слоны-в-национальном-парке.jpg`

3. **Запустите скрипт обработки:**
   ```bash
   cd backend
   node scripts/process-product-images-from-folder.js
   ```

4. Скрипт автоматически:
   - Найдет все изображения в папке
   - Оптимизирует их (ресайз до 1200px, сжатие)
   - Сохранит в `backend/uploads/products/`
   - Обновит ссылки в БД

### Вариант 3: Использование placeholder изображений

Если у вас пока нет изображений, можно временно использовать placeholder. Для этого:

1. Создайте или скачайте placeholder изображения
2. Поместите их в `frontend/public/products/` с правильными именами
3. Запустите скрипт обработки

## Проверка

После загрузки изображений проверьте:
1. Изображения должны быть в `backend/uploads/products/`
2. В БД поле `image_url` должно содержать путь вида `/uploads/products/...`
3. На сайте изображения должны отображаться
